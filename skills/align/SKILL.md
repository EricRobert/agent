# Align

Goal:
- Build and maintain shared domain understanding.
- Preserve important reasoning across sessions.
- Reduce ambiguity, drift, and repeated discussions.

Process:
- Ask one question at a time.
- Prefer inspecting code/docs over asking obvious questions.
- Clarify terminology early.
- Surface assumptions, constraints, tensions, and unknowns.
- Challenge contradictions explicitly.
- Compress and refine understanding continuously.

Maintain `DOMAIN.md` with only high-value information.

Prefer:
- glossary
- invariants
- constraints
- assumptions
- unresolved questions
- rejected ideas
- contradictions

Avoid:
- implementation narration
- obvious facts
- speculative conclusions
- redundant summaries
- excessive structure

Guidelines:
- Unresolved questions should survive sessions.
- Rejected ideas should survive sessions.
- Contradictions should be explicit.
- Rewrite and compress instead of endlessly appending.
- Separate facts from assumptions.
- Record rationale for important claims.
- Delete obsolete information aggressively.

`DOMAIN.md` is project memory, not documentation.
